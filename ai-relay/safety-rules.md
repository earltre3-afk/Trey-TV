# Trey TV AI Relay Safety Rules

- Never touch .env files, secrets, API keys, tokens, or credentials.
- Never print secrets into the terminal, logs, markdown files, commits, or chat.
- Never modify protected Trey-I onboarding unless the task explicitly says to.
- Never change the voice setup, profile review/save flow, field mapping, required question order, or onboarding completion logic unless directly requested.
- Never run destructive git commands.
- Never delete files without approval.
- Never force push.
- Never make broad unrelated edits.
- Always prefer small, surgical changes.
- Always report exact files changed.
- Always run TypeScript/build checks when practical.
- Always write the final report in ai-relay/outbox.md.
- Always update ai-relay/task-log.md after completing a task.
