- **120 errors** are **pre-existing in the parent's own code** (`feed/PostCard` `setBusyAction`, `features/games`, `features/prescribe-me`, `lib/tests/naturalAbilityStorage`, …) — unrelated to this port. The parent's build script is `vite build` (no typecheck), so the parent already ships with these.
- **106 errors** are in the ported `src/tradio` tree. **They are NOT port/codemod bugs** (zero `TS2307` module-resolution errors). They are the **Tradio source's own pre-existing type errors**, exposed here for the first time because:
  - The Tradio source `tsconfig.json` uses `"files": []` + project `references` and `strict:false`, so `npx tsc --noEmit` there checks **zero files** (a no-op). Prior "tsc clean" in the Tradio repo was a false green.

  - The parent uses `include:["src/**"]` + `strict:true` — the first genuine strict typecheck of this code → 106 surface (mostly `strictNullChecks`/`noImplicitAny`, plus a few real bugs e.g. missing `LegalRequestPriority`).
- The 106 are **compile-time only**; the transpiled runtime JS is the same code that ran in the standalone Tradio app, so `/tradio` renders.
